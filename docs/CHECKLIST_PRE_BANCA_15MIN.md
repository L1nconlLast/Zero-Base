# Checklist Pré-Banca (15 minutos) — Zero Base

## Bloco 1 — Identificação e contexto (2 min)
- [ ] Nome completo atualizado em todas as páginas acadêmicas.
- [ ] Nome do professor orientador preenchido.
- [ ] Data atualizada em todas as páginas.
- [ ] Título do projeto padronizado como **Zero Base**.

## Bloco 2 — Evidências obrigatórias (4 min)
- [ ] Link do repositório GitHub está correto e público.
- [ ] Links de commits citados nas páginas funcionam.
- [ ] Pelo menos 3 evidências visuais anexadas (dashboard, fluxo de estudo, resultados).
- [ ] Workflow/CI citado corresponde ao arquivo real do repositório.

## Bloco 3 — Métricas e método (4 min)
Para cada métrica apresentada:
- [ ] Existe método de coleta descrito (comando/ferramenta).
- [ ] Existe data da medição.
- [ ] Existe valor bruto (antes/depois quando aplicável).
- [ ] Existe evidência (print, log ou referência verificável).

Validação mínima recomendada:
- [ ] Contagem de linhas (ex.: `wc -l` ou equivalente).
- [ ] Tempo de build (ex.: `npm run build` com medição consistente).
- [ ] Indicador de qualidade (ex.: Lighthouse ou relatório equivalente).

## Bloco 4 — Qualidade acadêmica (3 min)
- [ ] Linguagem objetiva, sem tom promocional.
- [ ] Estrutura científica explícita: problema → metodologia → resultados.
- [ ] Limitações reais descritas com clareza.
- [ ] Trabalhos futuros priorizados e realistas.

## Bloco 5 — Ensaio final (2 min)
- [ ] Sequência de apresentação testada em 10 minutos.
- [ ] Transições entre páginas estão claras.
- [ ] Página de resultados aberta para perguntas da banca.
- [ ] Resumo executivo pronto para compartilhamento/entrega.

---

## Critério de pronto para banca
Considere pronto quando:
1. todos os itens acima estiverem marcados;
2. nenhuma métrica estiver sem método/evidência;
3. o roteiro couber em até 10 minutos com margem para perguntas.
