# Docs — Índice Vivo

## Estrutura oficial
- `01_product/` — visão do produto, fluxos e decisões funcionais
- `02_engineering/` — arquitetura, contratos, playbook, padrões
- `03_release/` — operação, deploy, rollback, checklists
- `04_research/` — pesquisa, validação, banca, entrevistas
- `99_archive/` — histórico morto ou quase morto

## Regra
Antes de criar um arquivo novo, classifique:
1. Produto
2. Engenharia
3. Release
4. Pesquisa
5. Arquivo morto

Se não entrar em uma dessas categorias, o arquivo não deve nascer em `docs/`.
